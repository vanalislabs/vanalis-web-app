#### Perbaiki Filter Function
```typescript
function filterByQuery<T extends ProjectSearchResult | DatasetSearchResult | UserSearchResult>(
  items: T[],
  query: string
): T[] {
  if (!query || query.length < 2) {
    return [];
  }

  const lowerQuery = query.toLowerCase();
  return items.filter((item) => {
    // Multi-field search based on type
    if ('title' in item) {
      // ProjectSearchResult
      const project = item as ProjectSearchResult;
      return (
        project.title.toLowerCase().includes(lowerQuery) ||
        project.curatorName.toLowerCase().includes(lowerQuery) ||
        project.status.toLowerCase().includes(lowerQuery) ||
        project.reward.toLowerCase().includes(lowerQuery)
      );
    } else if ('name' in item) {
      // DatasetSearchResult
      const dataset = item as DatasetSearchResult;
      return (
        dataset.name.toLowerCase().includes(lowerQuery) ||
        dataset.category.toLowerCase().includes(lowerQuery) ||
        dataset.price.toLowerCase().includes(lowerQuery)
      );
    } else if ('username' in item) {
      // UserSearchResult
      const user = item as UserSearchResult;
      return (
        user.username.toLowerCase().includes(lowerQuery) ||
        user.address.toLowerCase().includes(lowerQuery) ||
        user.roles.some(role => role.toLowerCase().includes(lowerQuery))
      );
    }
    return false;
  });
}
```

## Acceptance Criteria

1. **Mock Data Coverage:**
   - Minimal 15-20 items per kategori (projects, datasets, users)
   - Variasi yang cukup untuk testing berbagai search scenarios
   - Data realistis dan konsisten dengan domain AI/ML

2. **Search Functionality:**
   - Search bisa menemukan items berdasarkan multiple fields:
     - Projects: title, curatorName, status, reward
     - Datasets: name, category, price
     - Users: username, address, activity labels (roles field - yang sebenarnya adalah activity labels, bukan role terikat)
   - Partial matching bekerja dengan baik (contoh: "med" match "Medical")
   - Case-insensitive search bekerja dengan baik

3. **User Experience:**
   - Search results relevan dengan query
   - Tidak ada false positives atau false negatives yang signifikan
   - Loading state dan empty state bekerja dengan baik

4. **Performance:**
   - Filter function efisien untuk 15-20 items per kategori
   - Tidak ada lag saat mengetik di search input
   - Debounce tetap bekerja dengan baik
